## Architecture Highlights

### Data Flow
```
User Action → Event Handler → State Update → DOM Update → Visual Feedback
```

### Feed Loading Flow
```
Select Feeds → Check Cache → Render Cached → Fetch Fresh → Update if Changed
```

### State Management
- **Persistent**: localStorage (user preferences)
- **In-Memory**: Maps and arrays (runtime state)
- **Reactive**: State changes trigger UI updates

### Component Communication
- **Direct Calls**: Functions call each other directly
- **DOM**: State reflected in DOM, components read from DOM
- **Events**: Event delegation for dynamic content
