# Learning Path

1. Read `docs/learn/project-overview.md` for purpose and feature summary.
2. Review `docs/learn/project-structure.md` to understand where things live.
3. Study `docs/learn/architecture.md` to see data flow and module roles.
4. Read `docs/learn/key-components.md` for the main moving parts.
5. Check `docs/learn/features.md` to map UI features to code.
6. Review `docs/learn/configuration.md` for runtime state and endpoints.
7. Use `docs/learn/development-workflow.md` when running locally.
